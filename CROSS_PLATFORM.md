# Cross-platform notes — iCode camp repo

Repo is authored primarily on Windows; macOS work is supported. This doc
covers what differs across machines so neither platform pollutes the repo.

## Engine pin

- Godot **4.6.3** (stable). Match the minor version exactly across machines.
  Mismatched `.tscn`/`.tres` format versions cause silent re-saves that
  flood diffs.
- Project file declares `config/features=PackedStringArray("4.6", ...)`.

## What's tracked vs. ignored

Tracked (you should see these in `git status` after editing):
- `*.tscn`, `*.tres`, `*.gd`, `*.uid` — Godot source
- `*.import` — per-asset import settings + stable UIDs. **Critical**: if
  these are missing on a fresh clone, every `.tscn` reference re-resolves
  and UIDs drift between machines. They are small text files; commit them.
- `assets/**/*.glb`, `*.png`, etc. — source art
- `_balance/**` — Python sim/tuning code

Ignored (regenerated per machine):
- `.godot/` — import cache, shader cache. Per-machine, never commit.
- `dist/` — student ZIPs. Generated by the build script.
- `__pycache__/`, `*.pyc`, `.pytest_cache/`
- `.DS_Store`, `Thumbs.db`, `desktop.ini`

## Line endings

`.gitattributes` enforces `* text=auto eol=lf`. Every text file checks in
as LF regardless of platform. PowerShell scripts (`*.ps1`) keep CRLF
(Windows-only anyway). Binary assets are flagged so git never tries to
normalize them.

## Build script

`build/build_templates.py` is the canonical cross-platform build:

```bash
python build/build_templates.py                         # all days
python build/build_templates.py --day Day1_Pong_Game    # one day
```

Produces `dist/DayN_*_Template.zip` + `dist/DayN_*_Complete.zip` per day.
Strips `#@todo`/`#@end` marker blocks (BIBLE §11 C1 model). Requires
Python 3.10+ (uses PEP 604 type hints).

`build/build_templates.ps1` is the legacy Windows-only version. Kept for
parity until ZIPs from both have been diffed clean on one machine; then
the `.ps1` can be deleted.

## Platform-specific export

`.exe` (Windows) export requires the Windows export templates installed
in Godot on a Windows machine. `.app` (macOS) and `.zip` (HTML5/Linux)
likewise need their templates on their host platform. Source ZIPs from
`build_templates.py` work everywhere — they're just the project tree.

## First-time clone on a new machine

```bash
git clone <repo>
cd iCode
# Open Godot 4.6.3
# File → Open Project → DayN_<game>/project.godot
# Wait for "Importing assets..." to finish (regenerates .godot/ cache)
# F5 to run
```

If the import step fails with "UID conflict" or "missing resource":
- Confirm `.import` files are present (`git ls-files | grep .import | wc -l`)
- If zero, the gitignore is broken; check `.gitignore` for `*.import`
  (must NOT be there)

## Case-sensitivity

Default macOS APFS is case-insensitive, matching Windows. If your mac
uses case-sensitive APFS (rare), every `res://` path must match the
on-disk case exactly. Godot's editor enforces this on save, so editing
on case-sensitive macOS first surfaces any mismatches.
