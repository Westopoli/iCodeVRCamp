# Day 5 — Rally Camp (Instructor Notes)

Companion to BIBLE §6 "Day 5 — Racing" lock. D5 is **customization day** — no new code concepts, no Final Challenge. The race + tuning IS the engagement.

## One-time setup

1. Open Godot 4.6.x.
2. Import `Day5_Racing/project.godot`.
3. Press F5 to run.
4. **First launch reimports 112 Kenney `.glb` files — takes 30-60s.** Subsequent launches are instant. Let it finish before mashing F5 again.

## Game mechanics (instructor cheat sheet)

- **Viewport:** 1280×720. Forward+ renderer. Godot 4.6.3.
- **Genre:** Art-of-Rally-inspired 3D rally racer. Solo time trial + ghost replay.
- **Camera:** isometric chase — follows car position, not rotation. Default 20m back, 15m up, 45° down. `@export` knobs on CameraRig.
- **Win condition:** complete 3 laps → race-complete panel with total time + best lap. Kid can keep driving after.
- **Ghost replay:** first lap = no ghost shown. After completion, a semi-transparent overlay of the best lap replays alongside the live car on subsequent laps. Same mesh, `modulate = Color(1,1,1,0.4)`.
- **Drift:** binary handbrake. Power-slide sustains when throttle ≥ `throttle_powerslide_threshold` while handbrake held.

## Inputs

- **W / Up** — throttle
- **S / Down** — brake
- **A / Left** — steer left
- **D / Right** — steer right
- **Space** — handbrake
- **R** — reset car
- **P** — pause toggle
- **Esc** — quit to desktop

## Tuning

Car physics live in two places:

1. **`car_tune.json`** — 8 params (mass, engine_force, brake_force, max_steer_angle, front_grip, rear_grip, handbrake_grip_multiplier, throttle_powerslide_threshold). Generated by the offline sim. Kids can edit values in a text editor.
2. **3 `@export` wheel knobs on Car** — `suspension_stiffness`, `suspension_travel`, `damping_compression/relaxation`. Kid tunes via Godot Inspector while car is selected.

HUD displays speed + slide angle + lap times so kids/instructor can tune by feel.

## Sim-tuning origin

`car_tune.json` was generated **before camp** by `_balance/Day5/` — a Python bicycle-sim + random-search tuner. Per BIBLE D5: "sim ~80% + manual ~20% via Godot HUD."

- **Don't ship `_balance/` to kids.** The build script regex excludes folders by `^Day\d` filter, so `_balance` never gets zipped.
- Sim handles coarse balance (does the car turn at all? does it explode?). Manual tweaks in-engine handle feel.
- If a kid asks "where did these numbers come from?" — short answer: "A robot drove a fake car a thousand times and picked the numbers that felt best." Long answer: open `_balance/Day5/README.md` outside camp hours.

## Track customization (§10)

Two workflows, mix freely:

1. **GridMap painting** — kid paints custom tracks tile-by-tile with the MeshLibrary.
2. **Prefab drag-drop** — kid drags pre-built corner `.tscn` files from `prefabs/` into `Main.tscn → Track → StarterTrack`.

Available prefabs:

- `Hairpin_L.tscn` / `Hairpin_R.tscn` — tight U-turns
- `Sweeper_L.tscn` / `Sweeper_R.tscn` — wide arcs
- `Chicane.tscn` — S-bend
- `Straight_Short.tscn` / `Straight_Long.tscn`
- `Start.tscn` / `Finish.tscn` — gate prefabs
- `90_L.tscn` / `90_R.tscn` — medium right-angle corners
- `S_curve.tscn` — gentle S

Hybrid workflow per BIBLE D13 lock.

## Checkpoints

Three `Area3D` nodes (`Checkpoint1`, `Checkpoint2`, `Checkpoint3`) are pre-placed in `Main.tscn → Track`. Kid drags them to match their custom track layout.

**Lap counts only if crossed in order:** Start → 1 → 2 → 3 → Finish. Skip one and the lap won't register. This is intentional anti-cheese.

## Ghost replay

- First lap: ghost is hidden.
- On lap completion: car records `{t, pos, rot}` snapshots (~10-frame interval) into `lap_history`.
- If the just-finished lap is a new best, it becomes the ghost source.
- Subsequent laps: ghost replays at fixed rate, interpolating between snapshots. Same `raceCarRed.glb` mesh, transparent.

## Common kid stuck-points (anticipated)

1. **"My car flips on the first turn"** → suspension too soft. Bump `suspension_stiffness` (Car Inspector) from 50 → 80.
2. **"My car won't turn / understeers"** → `front_grip` too low. Edit `car_tune.json`, push from 1.118 toward 1.4.
3. **"My car won't drift"** → `handbrake_grip_multiplier` too high. Lower from 0.267 toward 0.15. Also confirm kid is mashing Space, not tapping.
4. **"My lap doesn't count!"** → missed a checkpoint. Open `Main.tscn`, find the offending Checkpoint Area3D, drag it to a spot the car *must* drive through.
5. **"My ghost is sitting still / never appears"** → no completed lap yet. Finish lap 1, ghost appears on lap 2.
6. **"Game won't launch — long pause"** → first-launch `.glb` reimport. Wait 30-60s.
